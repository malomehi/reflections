What happens when you initialize a repository? Why do you need to do it?
   Config files are created. It needs to be done in order to track the git operations in the repository.

How is the staging area different from the working directory and the repository? What value do you think it offers?
   Staging area are those files to be commited, not all the files in the working directory need to be in the staging area so we can decide what in the working directory go to the repository.

How can you use the staging area to make sure you have one commit per logical change?
   Adding to the staging area all the files corresponding to a logical change before performing the commit

What are some situations when branches would be helpful in keeping your history organized? How would branches help?
   When there is a significant change to not be included in the main project (extra functionalities, experimental features, etc.)

How do the diagrams help you visualize the branch structure?
   Diagrams show where the diversions happen
